Authentication

Micropay Bazaar uses API keys for authentication. Protected routes require your key in the Authorization header.

Demo key: mp_live_demo_key — use this for all hackathon testing.
Header
Authorization: Bearer mp_live_demo_key

Register a Service

POST/api/v1/registry/register

Registers an AI tool into the global Micropay Bazaar discovery index. Agents can dynamically discover your service and pay via x402 protocols.

Body Parameters
namestringrequired
The human-readable name of your service.
descriptionstringrequired
Detailed description used for agent semantic search.
endpointstringrequired
The fully qualified URL where your service API resides.
cost_usdnumberrequired
The exact cost per execution in USD.
developer_walletstringrequired
Your Solana base58 wallet address for settlement.
Request
curl -X POST https://micropay.up.railway.app/api/v1/registry/register \ -H "Authorization: Bearer mp_live_demo_key" \ -H "Content-Type: application/json" \ -d '{ "name": "Weather Oracle", "description": "Real-time meteorological data.", "endpoint": "https://api.example.com/weather", "cost_usd": 0.05, "developer_wallet": "2Hn6ES...xG9" }'
Response
{ "id": "srv_ak392nf9d21", "object": "service", "created_at": "2026-02-28T12:00:00Z", "name": "Weather Oracle", "cost_usd": 0.05 }

Discover Services

GET/api/v1/registry/discover

Returns all registered services. Unauthenticated. 26 services preloaded covering weather, finance, news, sports, AI, blockchain, and more. Use query to search by intent.

Query Parameters
querystring
Optional. Search by intent (e.g., 'weather', 'crypto', 'stocks').
Request
curl https://micropay.up.railway.app/api/v1/registry/discover?query=finance
Response
[ { "id": "finance_stocks", "name": "Stock Price Feed", "category": "Finance", "description": "Live equity prices from NYSE and NASDAQ.", "developer_wallet": "2Hn6ES...zv8" }, ... ]

Get a Service

GET/api/v1/registry/services/:id

Retrieves a single service by its ID. Use IDs from the discover endpoint (e.g., finance_stocks, weather_openmeteo).

Request
curl https://micropay.up.railway.app/api/v1/registry/services/finance_stocks

Create a Charge

POST/api/v1/charges

Initiates an x402 payment flow. Returns a Base64-encoded unsigned Solana transaction ready for Phantom to sign and broadcast. The developer_wallet is pulled automatically from the service registry.

Headers
Idempotency-Keystring
Optional. Prevents duplicate charges on retry.
Body Parameters
service_idstringrequired
ID of the service to consume (e.g., finance_stocks, weather_openmeteo).
source_walletstringrequired
Public Solana address of the wallet paying the charge.
amount_usdnumberrequired
Amount in USD (e.g., 0.05). Converted to SOL at live oracle price.
Request
curl -X POST https://micropay.up.railway.app/api/v1/charges \ -H "Authorization: Bearer mp_live_demo_key" \ -H "Content-Type: application/json" \ -d '{ "service_id": "finance_stocks", "source_wallet": "AuofYo21...Nt5", "amount_usd": 0.05 }'
Response
{ "id": "ch_wb4bedrhxh1q", "object": "charge", "status": "requires_signature", "service_id": "finance_stocks", "service_name": "Stock Price Feed", "amount_usd": 0.05, "exchange_rate_sol_usd": 150.0, "amount_sol": 0.000333, "source_wallet": "AuofYo21...Nt5", "destination_wallet": "2Hn6ES...zv8", "transaction_payload": "AQAAAAAAAAAA..." }

List Charges

GET/api/v1/charges

Returns a paginated list of charges, optionally filtered.

Query Parameters
limitint
Max records to return (1–100, default 10).
offsetint
Pagination offset.
statusstring
Filter by status: requires_signature, confirmed, failed.
source_walletstring
Filter by payer wallet address.
Request
curl "https://micropay.up.railway.app/api/v1/charges?limit=5" \ -H "Authorization: Bearer mp_live_demo_key"

Get a Charge

GET/api/v1/charges/:id

Retrieves the full details of a previously created charge by its ID.

Request
curl https://micropay.up.railway.app/api/v1/charges/ch_wb4bedrhxh1q \ -H "Authorization: Bearer mp_live_demo_key"

Chat Completions

POST/api/v1/chat/completions

OpenRouter proxy with Micropay Bazaar tools auto-injected. Agents use this to discover and invoke registry services dynamically. Compatible with the OpenAI Chat Completions API format.

Request
curl -X POST https://micropay.up.railway.app/api/v1/chat/completions \ -H "Authorization: Bearer mp_live_demo_key" \ -H "Content-Type: application/json" \ -d '{ "model": "gpt-4o-mini", "messages": [ {"role": "user", "content": "What is the price of AAPL?"} ] }'

Direct AI Chat

POST/api/ai/chat

Passthrough plain-text chat to the underlying AI layer.

Body Parameters
messagestringrequired
The text message for the assistant.
modelstring
Defaults to 'gpt-4o-mini'.
Request
curl -X POST https://micropay.up.railway.app/api/ai/chat \ -H "Authorization: Bearer mp_live_demo_key" \ -H "Content-Type: application/json" \ -d '{"message": "Summarize the Micropay x402 protocol"}'
