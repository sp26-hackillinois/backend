Quick Start

3 steps to your first payment

Micropay Bazaar turns any API call into a Solana micropayment. Your AI agent pays only for what it uses — no subscriptions, no credit cards, no minimums.

The x402 Payment Flow
1
POST /api/v1/charges
Send your service_id + source_wallet. Receive a Base64 unsigned Solana transaction.
2
Sign with Phantom (or keypair)
Decode transaction_payload, refresh blockhash, sign, and broadcast to Solana Devnet.
3
Access your data
Payment settles on-chain in under a second. Your agent receives the service response.

Registry

Register a Service

POST/api/v1/registry/register🔑 Auth required

Registers your API as a paid service in the discovery index. Once registered, AI agents can discover and call it, paying you in SOL per request via Phantom.

Body Parameters

namestringrequired
Human-readable name. Shown in discovery results and AI agent responses.
descriptionstringrequired
What your service does. Used for semantic search — write this like a tool description for an AI agent.
endpointstringrequired
Fully-qualified URL where your API lives.
cost_usdnumberrequired
Price per call in USD. Must be a positive number (e.g. 0.05).
developer_walletstringrequired
Your Solana wallet address (base58). Payments are sent here when agents call your service.

Returns

201 Created with the full service object including auto-generated id. Save this — you'll need it when creating charges.

Request cURL
curl -X POST https://micropay.up.railway.app/api/v1/registry/register \
  -H "Authorization: Bearer mp_live_demo_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Weather API",
    "description": "Real-time temperature, humidity, and wind speed for any city worldwide.",
    "endpoint": "https://my-api.com/weather",
    "cost_usd": 0.01,
    "developer_wallet": "2Hn6ESeMRqfVDTpt...zv8"
  }'
Response — 201 Created
{
  "id": "srv_a2k9xb3m7",
  "object": "service",
  "created_at": "2026-02-28T18:00:00.000Z",

  "description": "Real-time temperature...",

Charges

Create a Charge

POST/api/v1/charges🔑 Auth required

The core endpoint. Returns an unsigned Solana transaction representing payment for a registry service. The backend looks up the developer_wallet automatically from the registry — you never send money to yourself.

What the backend does automatically: fetches the live SOL/USD oracle price, calculates exact SOL amount, builds the transaction with correct feePayer and recentBlockhash, and returns it Base64-encoded. You just sign it.

Headers

Idempotency-Keystringoptional
A unique string (e.g. UUID) preventing duplicate charges on retry. Same key twice returns the cached result from the first call.

Body Parameters

service_idstringrequired
The registry service to pay for. Must exist. Examples: finance_stocks, weather_openmeteo, nlp_sentiment.
source_walletstringrequired
Base58 Solana public key of the wallet paying. The backend sets this as feePayer — Phantom must be connected with this exact wallet.

Charge Object Fields

FieldDescription
idUnique charge ID (prefix ch_).
statusAlways requires_signature on creation.
amount_usdFixed USD cost, determined by service category.
exchange_rate_sol_usdSOL/USD rate used at charge time (live oracle).
amount_solExact SOL to transfer.
network_fee_solSolana network fee: 0.000005 SOL.
destination_walletDeveloper's wallet — auto-fetched from registry.
transaction_payloadBase64 unsigned Solana tx. Decode → sign → broadcast.
Request cURL
curl -X POST https://micropay.up.railway.app/api/v1/charges \
  -H "Authorization: Bearer mp_live_demo_key" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: idemp_abc123" \
  -d '{
    "service_id": "finance_stocks",
    "source_wallet": "AuofYo21iiX8NQtgWBXLRFMiWfv83z2CbnhPNen6WNt5"
  }'
Response — 200
{
  "id": "ch_wb4bedrhxh1q",
  "object": "charge",
  "created_at": "2026-02-28T18:00:00.000Z",
  "status": "requires_signature",
  "service_id": "finance_stocks",
  "service_name": "Stock Price Feed",
  "amount_usd": 0.01,
  "exchange_rate_sol_usd": 148.32,
  "amount_sol": 0.000067423,
  "network_fee_sol": 0.000005,
  "source_wallet": "AuofYo21...Nt5",
  "destination_wallet": "2Hn6ESe...zv8",
  "transaction_payload": "AQAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAABAAQADb8d..."
}
Sign & send — JavaScript
// 1. Decode the payload
const tx = Transaction.from(
  Buffer.from(charge.transaction_payload, 'base64')
);

// 2. Refresh blockhash (prevents expiry)
const { blockhash, lastValidBlockHeight }
  = await connection.getLatestBlockhash('confirmed');
tx.recentBlockhash = blockhash;

// 3. Phantom signs (popup appears)
const signed = await phantom.signTransaction(tx);

// 4. Broadcast to Devnet
const sig = await connection.sendRawTransaction(
  signed.serialize()
);
// Explorer: https://explorer.solana.com/tx/${sig}?cluster=devnet

AI Chat Gateway

Chat Completions

POST/api/v1/chat/completions🔑 Auth required

The AI agent gateway. Send a natural language message — the AI decides whether to answer directly or request a paid tool call from the registry.

Two response types: type: "message" = plain AI answer. type: "tool_call" = AI needs paid data. If it's a tool_call, create a charge → sign → broadcast → call POST /chat/tool-result with the data.

Body Parameters

messagestringrequired
The user's message. Example: "What is the current price of AAPL?"
conversation_idstringoptional
Pass the conversation_id from a previous call to continue. Omit to start a new conversation (one is auto-generated). Persists up to 20 messages in server memory.
source_walletstringoptional
Informational only. Not used to build the transaction here.

Agentic Payment Flow

When the AI needs paid data
1
POST /chat/completions → type: "tool_call"
Response includes tool_call.service_id and estimated_cost_usd.
2
POST /charges → sign → broadcast
Create a charge with the service_id, sign with Phantom, broadcast to Devnet.
3
POST /chat/tool-result → type: "message"
Send conversation_id + the real data. AI returns a natural language answer.
Request cURL
curl -X POST https://micropay.up.railway.app/api/v1/chat/completions \
  -H "Authorization: Bearer mp_live_demo_key" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What is the current price of AAPL?",
    "source_wallet": "AuofYo21...Nt5"
  }'
Response A — Plain answer
{
  "type": "message",
  "conversation_id": "conv_a8f2k9x1b3",
  "content": "The x402 protocol enables...",
  "model": "gpt-4o-mini"
}
Response B — Tool call needed
{
  "type": "tool_call",
  "conversation_id": "conv_a8f2k9x1b3",
  "message": "I need to access Stock Price Feed. This requires a micro-payment of ~$0.01 in SOL on Devnet.",
  "tool_call": {
    "id": "call_abc123",
    "service_id": "finance_stocks",
    "service_name": "Stock Price Feed",
    "query": "What is the current price of AAPL?",
    "estimated_cost_usd": 0.01
  },
  "model": "gpt-4o-mini"
}
AI Chat Gateway

Submit Tool Result

POST/api/v1/chat/tool-result🔑 Auth required

After a tool_call response and successful payment, send the actual data back. The AI generates a natural language answer.

When to use: Only after receiving type: "tool_call" from /chat/completions, paying, and fetching the real data. Pass that data here as tool_result.

Body Parameters

conversation_idstringrequired
The exact conversation_id from the tool_call response. Must match to preserve context.
tool_resultstringrequired
The raw data from the service. JSON-stringified or plain text. The AI interprets and summarizes it.
service_idstringoptional
Fallback tool call ID if conversation history is missing one.
Request cURL
curl -X POST https://micropay.up.railway.app/api/v1/chat/tool-result \
  -H "Authorization: Bearer mp_live_demo_key" \
  -H "Content-Type: application/json" \
  -d '{
    "conversation_id": "conv_a8f2k9x1b3",
    "service_id": "finance_stocks",
    "tool_result": "{\"AAPL\":{\"price\":189.42,\"change\":\"+1.2%\",\"volume\":\"58M\"}}"
  }'
Response — 200
{
  "type": "message",
  "conversation_id": "conv_a8f2k9x1b3",
  "content": "Apple (AAPL) is currently trading at $189.42, up 1.2% on volume of 58M shares.",
  "model": "gpt-4o-mini"
}
AI Chat Gateway

Direct AI Chat

POST/api/ai/chat🌐 No auth

Simple pass-through to OpenAI. No tools, no registry, no conversation history. One-shot message → reply.

Body Parameters

messagestringrequired
The prompt to send to the AI.
modelstringoptional
One of: gpt-4o, gpt-4o-mini, o1-mini, o3-mini. Defaults to gpt-4o-mini.

Response Fields

replystring
The AI's response text.
modelstring
The model that was actually used.
usageobject
Token usage from OpenAI: prompt_tokens, completion_tokens, total_tokens.
Request cURL
curl -X POST https://micropay.up.railway.app/api/ai/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Explain the x402 micropayment protocol in 2 sentences.",
    "model": "gpt-4o-mini"
  }'
Response — 200
{
  "reply": "The x402 protocol is a standard for HTTP-native micropayments that lets APIs charge per request using blockchain transactions. When a client hits a paywalled endpoint, the server returns a transaction payload; the client signs and broadcasts it, then retries the request.",

  "usage": {
    "prompt_tokens": 22,
    "completion_tokens": 68,
